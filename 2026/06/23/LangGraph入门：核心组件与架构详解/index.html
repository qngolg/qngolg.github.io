LangGraph 入门：核心组件与架构详解

如果你已经用 LangChain 写过 RAG 或 Agent，大概率会遇到这样的困境：对话轮次一多，状态管理变得混乱；多 Agent 协作时，流程难以控制；关键操作需要人工审批，却没有优雅的暂停/恢复机制。

LangGraph 是 LangChain 团队推出的有状态图编排框架，专门用于构建可持久化、可中断、可观测的 Agent 工作流。它把 Agent 的执行过程建模为一张有向图：节点负责执行业务逻辑，边负责路由决策，全局 State 在各节点之间流转。

本文将系统梳理 LangGraph 的各个核心组件，并配合代码示例说明它们如何协同工作。

一、LangGraph 是什么？

在传统的 Chain 中，数据通常是单向流水线传递的：A 的输出作为 B 的输入，很难表达循环、分支和共享状态。

LangGraph 的核心思路是：

对比维度 传统 Chain LangGraph
状态管理 隐式传递 显式共享 State
控制流 线性为主 支持循环、条件分支
持久化 需自行实现 内置 Checkpointer
人工介入 难以实现 interrupt / Command 原生支持
可观测性 有限 支持 step 级 stream 与状态快照

典型应用场景包括：多轮对话 Agent、ReAct 工具调用、多 Agent 协作、审批流、长期运行的任务编排等。

3.2 Reducer：状态更新的合并策略

每个 State 字段可以绑定一个 Reducer，决定多个 Node 返回的局部更新如何合并到全局 State：

Reducer 类型 行为 典型场景
默认（无注解） 新值直接覆盖旧值 简单标量字段
add_messages 追加消息列表 对话历史
自定义 reducer 自定义合并逻辑 计数累加、去重合并等

Node 不需要返回完整 State，只需返回需要更新的字段（Partial Update），LangGraph 会按 Reducer 规则自动合并。

五、Node：业务逻辑的载体

Node 本质上是一个函数，签名为：

接收当前 State，返回需要更新的字段字典。

Node 设计建议

  1. 单一职责：每个 Node 只做一件事（调用 LLM、执行工具、格式化输出等）
  2. 返回 Partial：只返回变化的字段，不要构造完整 State
  3. 副作用隔离：数据库写入、API 调用等副作用应可重试；因为中断恢复后 Node 会从头重新执行
  4. 复杂逻辑拆分：一个 Node 内有多步操作时，可拆成多个 Node，便于调试和断点恢复

十二、Store：跨会话长期记忆

Checkpointer 保存的是单次 thread 内的状态历史；Store 则提供跨 thread 的长期记忆，例如用户偏好、知识库索引等。

在 Node 中可通过 get_store() 读写长期数据，实现「记住用户姓名」「跨对话积累经验」等能力。

十六、组件关系总结

组件 一句话理解
State 图的全局共享内存，带 Schema 和 Reducer
StateGraph 声明式图构建器
Node 执行业务逻辑的函数，State → Partial<State>
Edge 控制下一步走哪个 Node
Compile 构建器 → 可执行图，注入运行时配置
Checkpointer 按 thread 持久化状态，支持断点续跑
thread_id 会话隔离标识
Interrupt 暂停图执行，等待人工输入
Command 恢复执行、更新状态、跳转节点
Send 并行扇出多个子任务
Subgraph 子流程模块化复用
Store 跨会话长期记忆
Prebuilt ReAct Agent 等开箱即用模板

十七、实践建议

  1. **开发用 InMemorySaver，生产用 PostgresSaver**，不要在生产环境依赖内存 Checkpointer。
  2. Node 保持幂等：中断恢复会重跑整个 Node，避免不可重试的副作用。
  3. 先画状态图再写代码：理清 State 字段、Node 职责、边的条件，比直接堆代码更高效。
  4. **简单场景用 create_react_agent**，复杂编排再下沉到 StateGraph。
  5. **善用 stream_mode**：构建聊天 UI 时用 messages 模式做 token 流式输出。

总结

LangGraph 把 Agent 开发从「写 Prompt 链」升级为「设计状态机」。State 承载数据，Node 承载逻辑，Edge 承载决策，Checkpointer 赋予记忆，Interrupt / Command 打通人工审批，Subgraph / Send 支撑复杂编排。

掌握这些组件及其协作方式，你就能构建出生产级、可观测、可干预的 AI Agent 工作流。
